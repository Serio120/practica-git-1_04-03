Ejercicio 4: Comprobar el Estado del Repositorio
Preguntas a responder:
- ¿Qué comando usas para comprobar el estado del repositorio?
	- git status
- ¿Qué significa cuando Git dice que no hay archivos rastreados?
	 - Cuando Git dice que no hay archivos rastreados, significa que no hay archivos en el 		repositorio que estén siendo seguidos por Git para cambios. Los archivos no rastreados 	son 	aquellos que existen en el directorio de trabajo pero no han sido añadidos al índice       	(staging area). Para empezar a rastrear un archivo, debes usar el comando:
     		 git add nombre_del_archivo
      	Esto añadirá el archivo al índice, y Git comenzará a rastrear los cambios en ese archivo.

Ejercicio 6: Añadir un Archivo al Área de Preparación
Preguntas a responder: 
- ¿Qué comando usas para añadir un archivo al área de preparación?
	git add  
- ¿Qué significa añadir un archivo al área de preparación?
	Añadir un archivo al área de preparación (staging area) significa que estás marcando ese 	archivo y sus cambios para ser incluidos en el próximo commit. En otras palabras, estás 	preparando el archivo para que sus cambios sean registrados en el historial del repositorio 	cuando realices un commit.

- ¿Qué comando usas para realizar un commit? 
	git commit -m "mensaje"  (Por ejemplo crea el commit mensaje) 
- ¿Por qué es importante incluir un mensaje claro en cada commit?
	Incluir un mensaje claro en cada commit es importante por varias razones:
	Historial Comprensible:
	Colaboración Eficiente:
	Depuración y Resolución de Problemas:
	Documentación del Proyecto:
	Automatización y Herramientas:
	En resumen, mensajes de commit claros y descriptivos mejoran la mantenibilidad, colaboración 	y eficiencia en el desarrollo de software.

Ejercicio 9: Crear una Nueva Rama
Preguntas a responder:
- ¿Qué comando usas para crear una nueva rama en Git?
	git branch nombre_de_la_rama
- ¿Qué diferencia hay entre crear una rama y moverte a ella?
	Crear una rama y moverte a ella son dos acciones distintas en Git.
	Crear una rama solo establece una nueva rama en el repositorio, mientras que moverte a una 	rama cambia tu contexto de trabajo a esa rama.
